# 课表助手技术规格说明

## 1. 文档目标

本文基于 `PRD.md`，将产品需求转换为可开发、可测试、可验收的实现规格。当前实现优先覆盖 V1.0，预留 V1.1 上课地点导航的数据结构和扩展点。

## 2. 版本范围

### 2.1 V1.0 必做

- 面向学生的个人课表管理。
- 本地保存课程数据。
- 添加、编辑、删除课程。
- 按周查看课程安排。
- 支持上一周、下一周切换。
- 点击课程进入课程详情页。
- 展示作业、考试日期、备注。
- 同一时间课程冲突允许保存，但必须提示。
- 课程颜色自动分配，不支持用户自定义。

### 2.2 V1.0 不做

- 登录注册。
- 云端同步。
- 教务系统导入。
- 课程提醒。
- 单双周。
- 多学期管理。
- 课表分享。
- 用户自定义课程颜色。

### 2.3 V1.1 计划

- 接入高德地图 API。
- 根据教室字段解析地点。
- 在课程详情页提供一键导航。
- 地点解析失败不影响课程保存。

## 3. 用户与权限

V1.0 只面向学生，不需要登录账号，不区分角色权限。所有课程数据视为当前设备上的个人数据。

## 4. 核心数据模型

### 4.1 Course

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 课程唯一标识 |
| name | string | 是 | 课程名，建议不超过 30 个中文字符 |
| teacher | string | 否 | 任课教师，建议不超过 20 个中文字符 |
| classroom | string | 否 | 教室或上课地点，建议不超过 30 个字符 |
| weekday | number | 是 | 1-7，分别表示周一至周日 |
| startPeriod | number | 是 | 开始节次，范围 1-12 |
| endPeriod | number | 是 | 结束节次，范围 1-12，且不小于 startPeriod |
| startWeek | number | 是 | 开始周次，正整数 |
| endWeek | number | 是 | 结束周次，且不小于 startWeek |
| color | string | 是 | 系统自动分配的颜色值 |
| homework | string | 否 | 作业说明，多行文本 |
| examDate | string | 否 | 考试日期，建议使用 YYYY-MM-DD |
| note | string | 否 | 备注，多行文本 |
| locationName | string | 否 | V1.1 预留，高德地图地点名称 |
| locationAddress | string | 否 | V1.1 预留，高德地图地点地址 |
| longitude | number | 否 | V1.1 预留，经度 |
| latitude | number | 否 | V1.1 预留，纬度 |
| createdAt | string | 是 | 创建时间 |
| updatedAt | string | 是 | 更新时间 |

### 4.2 本地存储

- V1.0 使用本地存储保存课程数据。
- 存储内容至少包含课程列表和当前颜色分配结果。
- 数据读取失败时，应展示空课表或错误提示，不能导致页面不可用。
- 删除课程后，需要同步更新本地数据。

## 5. 业务规则

### 5.1 周次规则

- V1.0 只支持连续周，例如第 1-16 周。
- 当前查看周次在课程的 `startWeek` 和 `endWeek` 之间时，该课程展示在周视图。
- 当前查看周次不在课程周次范围内时，不展示该课程。

### 5.2 节次规则

- 默认每天 12 节。
- 课程节次必须满足 `1 <= startPeriod <= endPeriod <= 12`。
- 周视图纵向展示 1-12 节，可按课程跨度合并展示。

### 5.3 冲突规则

当两门课程满足以下条件时，视为课程冲突：

- weekday 相同。
- 当前周次与两门课程的周次范围均有交集。
- 节次范围有重叠。

冲突处理：

- 添加或编辑课程时允许保存。
- 保存前或保存后必须展示明显冲突提示。
- 周视图中冲突课程需要有可识别的冲突状态。

### 5.4 颜色规则

- 新增课程时系统自动分配颜色。
- 同一课程在不同周次保持相同颜色。
- V1.0 不支持用户手动选择颜色。
- 颜色需要保证课程块文字可读。

## 6. 页面规格

### 6.1 周视图页面

页面元素：

- 当前周次。
- 上一周按钮。
- 下一周按钮。
- 添加课程按钮。
- 星期列：周一至周日。
- 节次行：第 1 节至第 12 节。
- 课程块。
- 空状态。
- 冲突提示。

交互规则：

- 默认进入当前周。
- 点击上一周 / 下一周后更新当前周课程。
- 点击课程块进入课程详情页。
- 当前周无课程时展示空状态。

### 6.2 添加 / 编辑课程页面

字段：

- 课程名。
- 教师。
- 教室。
- 星期。
- 节次。
- 周次。
- 作业。
- 考试日期。
- 备注。

交互规则：

- 必填字段未填时禁止保存，并展示错误提示。
- 编辑保存后，周视图和课程详情页需要同步更新。
- 编辑失败时保留用户已填写内容。
- 修改时间或周次后，需要重新计算课程展示位置和冲突状态。

### 6.3 课程详情页

展示内容：

- 课程名。
- 教师。
- 教室。
- 星期。
- 节次。
- 周次。
- 作业。
- 考试日期。
- 备注。

操作：

- 返回周视图。
- 编辑课程。
- 删除课程。

空状态：

- 教师为空：显示“暂无教师信息”。
- 教室为空：显示“暂无教室信息”。
- 作业为空：显示“暂无作业”。
- 考试日期为空：显示“暂无考试日期”。
- 备注为空：显示“暂无备注”。

### 6.4 V1.1 上课地点导航扩展

V1.1 在课程详情页增加导航入口：

- 地点解析成功时展示导航入口。
- 点击导航后打开高德地图路线规划或地点页面。
- 用户拒绝定位权限时，仍可打开目标地点。
- 地点解析失败时，保留原始教室文本，不提供导航或提示地点不明确。

## 7. 校验与错误处理

| 场景 | 处理 |
| --- | --- |
| 课程名为空 | 禁止保存，提示“请输入课程名” |
| 星期未选择 | 禁止保存，提示“请选择星期” |
| 节次未选择 | 禁止保存，提示“请选择节次” |
| 周次为空或格式错误 | 禁止保存，提示“请输入有效周次” |
| 起始节次大于结束节次 | 禁止保存，提示“请输入有效节次” |
| 起始周次大于结束周次 | 禁止保存，提示“请输入有效周次” |
| 教师为空 | 允许保存 |
| 教室为空 | 允许保存 |
| 作业为空 | 允许保存 |
| 考试日期为空 | 允许保存 |
| 备注为空 | 允许保存 |
| 课程时间冲突 | 允许保存，但展示明显冲突提示 |
| 删除课程 | 必须二次确认 |

## 8. 非功能规格

- 周视图切换响应时间建议小于 300ms。
- 添加、编辑、删除课程后页面需要即时反馈。
- 优先适配移动端。
- Web 版本兼容主流现代浏览器。
- 桌面端需要支持更宽的周课表布局。
- 本地数据保存成功后不能丢失。

## 9. 架构建议

### 9.1 推荐技术栈

正式开发建议采用以下技术栈：

| 分类 | 选型 | 说明 |
| --- | --- | --- |
| 前端框架 | Vue 3 | 适合表单、列表、状态驱动的中小型业务工具 |
| 构建工具 | Vite | 启动快，配置轻，适合快速开发和演示 |
| 开发语言 | TypeScript | 提升课程数据模型、表单校验和业务规则的类型安全 |
| 路由 | Vue Router | 管理周视图、课程表单、课程详情等页面 |
| 状态管理 | Pinia | 管理课程列表、当前周次、选中课程等全局状态 |
| 样式方案 | 普通 CSS 或 CSS Modules | 当前界面规模不大，暂不引入复杂 UI 框架 |
| 本地存储 | localStorage | V1.0 数据量小，优先简单落地 |
| 后续存储扩展 | IndexedDB | 课程数据、学期数据、离线数据变复杂后再迁移 |
| 地图能力 | 高德地图 JS API / Web 服务 API | V1.1 支持地点解析和课程详情页一键导航 |

### 9.2 选型理由

- 课表助手属于轻量业务工具，核心是表单录入、列表展示、详情查看、本地状态管理和少量业务规则。
- Vue 3 的模板语法和双向绑定适合课程表单、周次切换、详情页展示等场景，开发效率高。
- TypeScript 可以约束 `Course` 数据结构，降低节次、周次、地点字段在页面间传递时的出错概率。
- Pinia 可以集中管理课程数据和当前周次，避免页面组件之间直接互相传状态。
- V1.0 数据保存在本地，使用 localStorage 足够；当后续支持多学期、导入、同步或大量课程数据时，再迁移到 IndexedDB 或后端服务。
- V1.1 地图能力应封装在独立服务中，避免页面组件直接依赖高德地图 API 细节。

### 9.4 版本演进建议

- V1.0：使用 Vue 3 + Vite + TypeScript + Pinia + localStorage，完成个人课表闭环。
- V1.1：接入高德地图 API，完成地点解析和课程详情页一键导航。
- V1.2：当数据复杂度上升后，再考虑 IndexedDB、云同步、登录和多学期管理。

### 9.5 模块划分

- Course Domain：课程实体、校验、冲突检测、周次过滤。
- Storage：本地数据读写、数据迁移。
- Schedule View：周视图布局、课程块展示、周次切换。
- Course Form：添加和编辑课程。
- Course Detail：课程详情、删除确认。
- Location Adapter：V1.1 高德地图解析与导航跳转。

### 9.6 关键函数建议

- `validateCourse(input)`：校验课程表单。
- `getCoursesForWeek(courses, week)`：按周过滤课程。
- `detectConflicts(courses, targetCourse)`：检测课程冲突。
- `assignCourseColor(existingCourses)`：自动分配课程颜色。
- `saveCourses(courses)`：保存本地课程数据。
- `loadCourses()`：读取本地课程数据。
- `buildAmapNavigationUrl(course)`：V1.1 构建高德地图跳转地址。

## 10. 待确认事项

以下事项未在 PRD 中完全明确，建议开发前确认：

| 编号 | 待确认事项 | 建议默认值 |
| --- | --- | --- |
| Q-001 | 当前周如何计算？是按自然周、开学第几周，还是用户手动设置当前教学周？ | 建议 V1.0 让用户手动设置或默认从第 1 周开始切换 |
| Q-002 | 学期总周数上限是多少？ | 建议默认 20 周 |
| Q-003 | 周视图是否需要展示周六、周日？ | PRD 已写周一至周日，建议保留 7 天 |
| Q-004 | 冲突提示是在保存前弹窗确认，还是保存后在周视图标记？ | 建议保存前确认，同时周视图标记 |
| Q-005 | 本地保存使用浏览器 localStorage、IndexedDB，还是移动端本地数据库？ | Web MVP 建议 localStorage，后续可迁移 |
| Q-006 | 是否需要数据导出或备份？ | V1.0 暂不做 |
| Q-007 | V1.1 高德地图 API Key 的申请、配置和安全策略由谁负责？ | 建议通过环境变量或配置文件注入 |
