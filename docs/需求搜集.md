# 课表助手需求搜集文档

## 1. 项目背景

课表助手是一款面向学生或教师的课程管理工具，用于集中记录、查看和维护每周课程安排，并支持查看课程相关的作业、考试日期和备注信息。

当前阶段优先完成核心课表功能，保证用户可以完成“录入课程、查看周课表、查看课程详情”的基础流程。

## 2. 产品目标

- 帮助用户快速添加、编辑和删除课程信息。
- 以周视图清晰展示课程安排，降低查课成本。
- 通过课程详情页集中展示课程相关事项，包括作业、考试日期和备注。
- 用不同颜色区分不同课程，提高课表识别效率。
- 后续通过高德地图 API 支持上课地点导航，降低用户找教室和赶课成本。

## 3. 目标用户

### 3.1 学生

- 需要管理个人每周课程安排。
- 需要查看课程地点、教师、节次和上课周次。
- 需要记录课程作业、考试时间和备注。
- V1.0 只面向学生，不支持教师 / 助教角色权限。

### 3.2 教师或助教

- 需要维护自己负责课程的上课时间和地点。
- 需要记录课程相关提醒信息。

## 4. 使用场景

### 4.1 新学期录入课表

用户在新学期开始时，根据学校课表逐门添加课程，填写课程名、教师、教室、节次和周次。

### 4.2 每周查看课程安排

用户打开课表助手后，默认查看当前周课表，快速确认每天有哪些课程、在哪个教室、由哪位教师授课。

### 4.3 临时修改课程信息

当课程教室、教师或上课周次发生变化时，用户可以进入课程编辑页修改课程信息。

### 4.4 查看课程详情

用户点击周视图中的某一门课程，进入课程详情页，查看该课程的作业、考试日期和备注。

### 4.5 删除无效课程

当课程结束或录入错误时，用户可以删除课程，删除后该课程不再出现在周视图中。

### 4.6 前往上课地点

用户在课程详情页查看上课地点时，可以点击导航入口，打开高德地图并以前往该课程地点为目的。该功能适合用户不熟悉教学楼位置、跨校区上课、或课间需要快速确认路线的场景。

## 5. 功能范围

### 5.1 课表管理

### 功能描述

用户可以添加、编辑、删除课程。课程是课表助手的核心数据对象。

### 课程字段

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 课程名 | 是 | 例如：高等数学、大学英语 |
| 教师 | 否 | 任课教师姓名 |
| 教室 | 否 | 上课地点，例如：教学楼 A101 |
| 地点名称 | 否 | 高德地图匹配到的标准地点名称 |
| 地点经纬度 | 否 | 高德地图匹配到的地点坐标 |
| 星期 | 是 | 周一至周日 |
| 节次 | 是 | 课程所在节次，例如：第 1-2 节 |
| 周次 | 是 | 上课周范围，例如：第 1-16 周 |
| 颜色 | 否 | 系统自动分配或用户手动选择 |
| 作业 | 否 | 课程相关作业记录 |
| 考试日期 | 否 | 课程考试时间 |
| 备注 | 否 | 其他补充说明 |

### 添加课程

用户可以通过“添加课程”入口创建新课程。

添加课程时至少需要填写：

- 课程名
- 星期
- 节次
- 周次

### 编辑课程

用户可以修改已添加课程的全部字段。

编辑保存后：

- 周视图中的课程信息同步更新。
- 课程详情页中的信息同步更新。

### 删除课程

用户可以删除已添加课程。

删除前需要二次确认，避免误删。

删除后：

- 周视图不再显示该课程。
- 课程详情页不可再访问该课程。

### 规则说明

- 同一时间段允许暂时存在课程冲突，但需要给出冲突提示。
- 周次字段需要支持连续周，例如第 1-16 周。
- V1.0 只支持连续周，不支持单双周。
- V1.0 默认每天 12 节。
- 课程颜色由系统自动分配，不支持用户自定义。
- 教室字段填写后，可通过高德地图 API 尝试解析地点。
- 地点解析失败不影响课程保存，系统需要保留用户原始填写的教室文本。
- 后续可扩展支持单双周、调课和停课。

### 5.2 周视图展示

### 功能描述

用户可以按周查看课程安排。周视图是课表助手的主页面。

### 展示内容

每个课程块至少展示：

- 课程名
- 教室
- 教师
- 节次

### 展示规则

- 横向按星期展示：周一至周日。
- 纵向按节次展示。
- 不同课程使用不同颜色区分。
- 当前周有课的课程显示在对应日期和节次位置。
- 当前周不上课的课程不显示在周视图中。

### 交互规则

- 点击课程块，进入课程详情页。
- 可切换上一周、下一周。
- 默认展示当前周。
- 当某天无课程时，显示空状态。

### 颜色规则

- 新增课程时系统自动分配颜色。
- 同一门课程在不同周次中保持相同颜色。
- 颜色应保证文字可读性。

### 5.3 课程详情页

### 功能描述

用户点击课程后进入详情页，查看课程完整信息和学习事项。

### 页面内容

课程详情页需要展示：

- 课程名
- 教师
- 教室
- 星期
- 节次
- 周次
- 作业
- 考试日期
- 备注

### 操作能力

课程详情页需要提供：

- 编辑课程入口
- 删除课程入口
- 返回周视图入口

### 作业展示

当前阶段作业可以作为文本字段记录。

示例：

```text
完成第 3 章课后习题，周五前提交。
```

### 考试日期展示

考试日期支持选择具体日期。

如暂未填写，显示“暂无考试日期”。

### 备注展示

备注用于记录其他课程相关信息。

如暂未填写，显示“暂无备注”。

### 5.4 上课地点导航

### 功能描述

用户可以在课程详情页点击导航入口，打开高德地图并前往课程对应的上课地点。

### 使用前提

- 课程已填写教室或上课地点。
- 系统可以通过高德地图 API 将教室文本解析为可导航地点。

### 功能规则

- 添加或编辑课程时，系统可根据教室字段进行地点解析。
- 地点解析成功后，保存地点名称和经纬度。
- 地点解析失败时，课程仍可保存，但不提供可用导航入口。
- 如果匹配到多个地点，用户可以选择一个地点；未选择时仅保存原始教室文本。
- 点击导航后，系统打开高德地图路线规划或地点页面。
- 用户拒绝定位权限时，系统仍可打开目标地点，但不使用当前位置作为起点。

### 暂不包含

- 校园室内导航
- 实时位置共享
- 自动迟到提醒
- 课间路线风险提示
- 教学楼地图管理后台

## 6. 用户流程

### 6.1 添加课程流程

1. 用户进入周视图。
2. 点击“添加课程”。
3. 填写课程信息。
4. 点击保存。
5. 系统校验必填项。
6. 保存成功后返回周视图。
7. 新课程显示在对应星期和节次中。

### 6.2 查看课程详情流程

1. 用户进入周视图。
2. 点击某个课程块。
3. 系统打开课程详情页。
4. 用户查看课程基础信息、作业、考试日期和备注。

### 6.3 编辑课程流程

1. 用户进入课程详情页。
2. 点击“编辑”。
3. 修改课程字段。
4. 点击保存。
5. 系统校验并保存。
6. 返回课程详情页或周视图。

### 6.4 删除课程流程

1. 用户进入课程详情页或课程编辑页。
2. 点击“删除课程”。
3. 系统展示删除确认弹窗。
4. 用户确认删除。
5. 系统删除课程并返回周视图。

### 6.5 上课地点导航流程

1. 用户进入课程详情页。
2. 系统判断课程是否存在已解析地点。
3. 如果存在已解析地点，展示导航入口。
4. 用户点击导航入口。
5. 系统打开高德地图，并将课程地点作为目的地。
6. 如果无法获取当前位置，只打开目标地点或路线规划页面。

## 7. 数据校验需求

| 校验项 | 规则 |
| --- | --- |
| 课程名 | 必填，建议不超过 30 个中文字符 |
| 星期 | 必选，范围为周一至周日 |
| 节次 | 必选，需要符合系统节次范围 |
| 周次 | 必填，需要为有效周次 |
| 教师 | 可选，建议不超过 20 个中文字符 |
| 教室 | 可选，建议不超过 30 个字符 |
| 地点名称 | 可选，需要来自地点解析结果或为空 |
| 地点经纬度 | 可选，需要为有效经纬度 |
| 考试日期 | 可选，需要为有效日期 |
| 作业 | 可选，支持多行文本 |
| 备注 | 可选，支持多行文本 |

## 9. 非功能需求

### 9.1 易用性

- 添加课程流程应尽量简短。
- 周视图应直观展示课程位置。
- 课程颜色需要清晰区分且不影响阅读。

### 9.2 性能

- 周视图切换应快速响应。
- 课程数量较多时，页面仍应保持流畅。

### 9.3 数据可靠性

- V1.0 课程数据保存在本地。
- 保存课程后数据不能丢失。
- 删除课程前必须二次确认。
- 编辑课程失败时，需要保留用户已填写内容。

### 9.4 兼容性

- 优先支持移动端使用。
- Web 版本需要适配常见桌面浏览器。
- 上课地点导航需要兼容移动端和桌面端浏览器，不强制要求安装高德地图 App。

## 10. 验收标准

### 10.1 课表管理验收标准

- 用户可以成功添加一门课程。
- 用户可以编辑已添加课程的任意字段。
- 用户可以删除已添加课程。
- 必填字段为空时不能保存，并显示提示。
- 删除课程前会出现确认提示。

### 10.2 周视图验收标准

- 系统可以按周展示课程。
- 不同课程显示为不同颜色。
- 同一课程颜色保持一致。
- 点击课程块可以进入对应课程详情页。
- 切换上一周、下一周后，课程展示正确。

### 10.3 课程详情页验收标准

- 课程详情页展示课程完整信息。
- 作业、考试日期、备注可以正常展示。
- 未填写作业、考试日期或备注时，有明确空状态。
- 用户可以从详情页进入编辑流程。
- 用户可以从详情页删除课程。

### 10.4 上课地点导航验收标准

- 教室填写后，系统可以尝试解析上课地点。
- 地点解析成功时，课程详情页展示导航入口。
- 用户点击导航入口后，可以打开高德地图并以课程地点作为目的地。
- 地点解析失败时，课程仍可保存，并保留原始教室文本。
- 教室为空时，课程详情页不展示可点击的导航入口。

## 13. 当前版本范围

### V1.0 必做功能

- 课表管理：添加、编辑、删除课程。
- 周视图展示：按周展示课表，不同课程颜色区分。
- 课程详情页：查看课程详情，包括作业、考试日期和备注。
- 本地保存课程数据。
- 默认每天 12 节，只支持连续周。

### V1.0 暂不包含

- 登录注册
- 云端同步
- 课程提醒
- 教务系统导入
- 课表分享
- 多学期管理
- 单双周
- 用户自定义课程颜色

### V1.1 计划功能

- 上课地点导航：接入高德地图 API，在课程详情页提供上课地点导航能力。
